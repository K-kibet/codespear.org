Java Documentation: Variables & Data Types

Introduction

Java is a static programming language. Static programming languages perform "type-checking" at compilation time. This is a contrast to dynamic programming languages like Javascript and Python that perform type-checking at run-time.

Type-checking is done by a type-checker, and it insures that the use of a construct (variables, arrays etc.) matches what is expected in its intended usage. For example, you would not expect a sentence (String Data Type) to perform a multiplication with a number (Integeral Data Type). Since Java performs these checks at compile time, it ensures that there are no type-errors during run time.

In Java there are 8 Primitive data types. These are boolean, chars, integeral data types and floating point data types. (These will be explained individually in greater detail later on.) Primitive types are the building blocks of data manipulation, and can be used to form Composite data types like Strings.

Declaring Variables

Declaring variables in Java is rather simple. You must declare the data type of the variable, its identifier also known as its name, and it must end with a semi-colon. It is recommended that you set an initial value for the variable. Various examples are included below:

int myNum = 9;
String aString = "Hello World";
boolean isValid = false;
char aChar = 'a';
double myDouble = 2.5;

It is possible, however, to declare a variable without providing an initial value like so:

int myNum;

Here are some rules and tips about variables:

  • Two variables cannot have the same name. It will return an error if you have an int variable called "myNum" and a double also called "myNum".
  • Capitilisation counts. "myNum" is not the same as "MYNUM" - They will be two separate variables.
  • Once you declare a variable, it is no longer necessary to type the variable type when calling the variable. E.g.:
    int myNum;
    myNum = 13;
  • The standard procedure for naming variables is to use camelCase. In camelCase, the first letter of every word is capitilised bar the first word. camelCase is used because it helps programmers read long variable names.
  • It is better to be specific with variable names. Choosing generic names can be very confusing in the long-run when you have dozens of different variables with similar names.

Integer Data Types

Integers can hold whole numbers like 4 or 126. There are four kind of integers, and the range of numbers they can store differs. This difference is based on the amount of bytes a number requires to be stored.

Type Size Range of values
byte 1 byte -128 to 127
short 2 bytes -32768 to 32767
int 4 bytes −2,147,483,648 to 2,147,483,647
long 8 bytes -9,223,372,036,854,775,808 to 9,223,372,036,854,755,807

"int" is the most commonly used integer. It is only in rare cases that int won't have the number you need to store. However, you are certain of a variable's value because it is a fixed value - it can be beneficial to use a smaller unit like a short or byte. This will conserve memory.

Floating Point Data Types

Floating Point data types are used to represent decimals and fractions in Java. There are two floating point types, floats and doubles. Like integers, each type can store a different range of values. Here are some examples of floating point types:

float myFloat = 5.5;
double myDouble = 667.6667;

As stated in the Integer section, it is good to use the 'smaller' type, if you are sure the value will be within its range to converse memory space. In floating points, the float is the smaller value.

Type Size Range of values
float 4 bytes 3.4e−038 to 3.4e+038
double 8 bytes 1.7e−308 to 1.7e+038

Boolean

The boolean data type has only two possible values: true or false. This data type is often used for conditional statements like if/else statements, while loops and for loops. E.g.:

boolean isValid = true;
if(isValid) {
 /* some code */
}
else {
 /* some code */
}

Chars

Chars are individuals characters and to assign a char to a char variable, you must enclose it in single quotes. here are a total of 65,535 possible char values. It has a minimum value of ‘u0000’ (or 0) and a maximum value of ‘uffff’ (or 65,535, inclusive). For every character there is a unicode equivalent. For example, as previously stated, '0' is the same as 'u0000'. To use this in Java, you must prefix the unicode value with a backslash. The full list of unicode characters can be found at this site. Here are some char examples:

char aChar = 'Z';
char aCharTwo = '\u20AC'; //The unicode for the Euro symbol

Strings

As stated previously, Strings are a composite data type, unlike all the other types that have been discussed so far. It is made up of a sequence of characters. Since it's a composite data type, Strings come from a class - the String class - and every String is an object of this class. String objects are immutable, which means that once created, their values cannot be changed unless you assign it a new value. The String class is implicit so you do not need to import it. Unlike chars, Strings are enclosed in double quotes like so:

String stringOne = "Hello World";
String stringTwo = new String("I am also a String!");
